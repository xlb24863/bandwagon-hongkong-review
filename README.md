## 四、三网路由回程分析：CN2 GIA 全程兜底

光看延迟不够，路由走的是哪条线才是关键。香港机房最怕的就是去程 CN2、回程绕普通骨干网，结果晚高峰一塌糊涂。搬瓦工 HKHK_8 的核心卖点就是三网强制回程 CN2 GIA（AS4809），这点在 NextTrace 路由追踪里看得清清楚楚。

**电信回程**：全程走 CTGNet → CN2-BackBone（59.43 开头 IP 段），跳数少，延迟低，直连广州/上海/北京电信节点。这是 CN2 GIA 最标准的回程路径。

**联通回程**：大部分城市走 CTGNet → CN2-BackBone（59.43 段）→ 4837 骨干网落地，少数城市（如北京联通部分线路）会接 CMI 走 9808 落地。整体延迟可控。

**移动回程**：走 CMI（AS58453）→ CMNET（AS9808）落地，部分城市会在中途借道 CN2-Global 做 BGP 优化。这是 HKHK_8 唯一稍微"打折"的部分——移动用户回程并非纯 CN2 GIA，而是 CMI + CN2 混合，所以延迟比电信联通稍高，但仍属优化线路。

> 简单说，HKHK_8 真正的"三网 CN2 GIA"指的是电信、联通走 CN2 GIA，移动走 CMI+CN2 混合。这在香港 VPS 里已经是顶级配置，市面绝大多数香港 VPS 要么纯 CMI，要么纯 PCCW，能做到电信联通双 CN2 GIA 的寥寥无几。
